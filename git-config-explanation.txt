## Git 配置命令失败原因分析

### 问题现象
在终端中执行 `git config -g user.email "baizerenyou@gmail.com"` 命令时出现错误：
```
error: unknown switch `g'
```

### 原因分析
Git 的 `config` 命令中，**全局配置的正确参数是 `--global`（双横线）**，而不是 `-g`（单横线）。

从 Git 帮助文档可以看到，`config` 命令支持的参数格式：
```
Config file location
    --[no-]global         use global config file
    --[no-]system         use system config file
    --[no-]local          use repository config file
```

### 当前状态
从 `.git/config` 文件可以看到：
- 本地仓库已成功初始化
- 本地用户配置已正确设置：
  ```
  [user]
      name = Isteyft
      email = baizerenyou@gmail.com
  ```

### 解决方法
如需设置全局 Git 配置，应使用正确的参数格式：
```bash
git config --global user.email "baizerenyou@gmail.com"
```

### 备注
- 本地配置（无参数）只对当前仓库有效
- 全局配置（`--global`）对当前用户的所有仓库有效
- 系统配置（`--system`）对系统所有用户有效
